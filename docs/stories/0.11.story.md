## Story

**Como** owner do evento TFTEC,
**eu quero** que as bandeiras das seleções sejam servidas do próprio domínio da app (não de `flagcdn.com`),
**para que** a geração de PDF do ingresso (html2canvas) não corra risco de falhar por canvas-taint/CORS se o CDN externo estiver lento ou indisponível durante o evento — fechando o único risco residual do AC-8.

---

## Contexto

Code review do AC-8 (Story 0.10) encontrou: `TicketStub.tsx` renderiza bandeiras com `<img src={flagcdn.com/...} crossOrigin="anonymous">`. Se o flagcdn falhar CORS/disponibilidade no momento do `html2canvas.toDataURL`, o canvas fica *tainted* e a exportação de PDF **lança e aborta**. O QR não é afetado (div puro), mas o ingresso inteiro falha.

Precedente: os 16 estádios já foram self-hosted (commit `b646073`, `public/stadiums/`). Aplicamos o mesmo padrão às 48 bandeiras.

**Decisão de design — rewrite no render, não no DB:** o seed/DB continua com URLs flagcdn (mudar o DB acoplaria ao problema bacpac/migration documentado no EPIC-001). O frontend reescreve `flagcdn.com/w80/{cc}.png` → `/flags/{cc}.png` em tempo de render. Zero mudança de dado em prod, independente do bacpac.

---

## Acceptance Criteria

1. **AC-1** — As 48 bandeiras (`flagcdn.com/w80/{cc}.png`) baixadas para `Lovable/World Cup Tickets Hub/public/flags/{cc}.png`.
2. **AC-2** — Helper `localFlag(url: string): string` que extrai o country code de uma URL flagcdn e retorna `/flags/{cc}.png`; passa URLs não-flagcdn/vazias inalteradas (fallback seguro).
3. **AC-3** — `TicketStub.tsx` (crítico p/ PDF) usa `localFlag()` nas duas `<img>` de bandeira. `crossOrigin` pode ser removido (same-origin).
4. **AC-4** — Componente compartilhado `TeamFlag.tsx` usa `localFlag()` (cobre as páginas que renderizam via `<TeamFlag>`). **Escopo honesto:** ~15 `<img>` diretos em 7 páginas (Matches, Teams, Qualified, TeamDetail, StadiumDetail, WorldCupDetail, WorldCupHistory) **NÃO** convertidos nesta story — são render de página (bandeira quebrada = cosmético, não falha funcional; app já opera com eles hoje). Deferido como follow-up de baixa prioridade (TD) para evitar gold-plating/regressão sob escopo de remediação.
5. **AC-5** — Build do frontend OK (`npm run build`); 48 arquivos presentes no `dist/flags/`.
6. **AC-6** — Smoke local: servir o `dist/` de produção, abrir um ingresso, gerar PDF → bandeiras aparecem, PDF não falha (valida o fix do risco).
